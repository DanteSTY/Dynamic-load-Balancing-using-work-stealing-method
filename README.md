#  Реализация динамической балансировки загрузки (Dynamic Load Balancing) методом "Work Stealing" в гибридной модели программирования (MPI + Pthreads)



**Дано:**
- Множество узлов (процессов MPI), соединенных сетью.
- Пул задач: Большое количество независимых задач (Jobs), время выполнения которых заранее неизвестно или сильно варьируется.
- Начальное распределение: Задачи распределены неравномерно (например, все задачи только у процесса с рангом 0, или у одних процессов "тяжелые" задачи, у других — "легкие").


**Проблема:**
При статическом распределении (поделить поровну в начале) возникает дисбаланс загрузки. Процесс А может закончить свои задачи за 5 секунд и простаивать, пока Процесс Б будет работать еще час. Это приводит к неэффективному использованию ресурсов кластера.




**Требуется реализовать:**
Систему, в которой простаивающие процессы активно ищут и забирают работу у загруженных соседей.




**Технические требования (Архитектура решения):**
Для максимальной эффективности каждый MPI-процесс должен быть распараллелен на потоки (Threads), чтобы коммуникация не блокировала вычисления:
* Поток-Worker (Рабочий): Занимается полезной нагрузкой (вычислениями). Не должен простаивать.
* Поток-Fetcher (Добытчик): Просыпается, когда у Worker'а заканчиваются задачи, и отправляет запросы другим узлам MPI с целью получить новую порцию работы.
* Поток-Responder (Ответчик): Фоновый поток, который слушает сеть. Если другой узел просит работу, Responder должен мгновенно отдать часть задач из локальной очереди, не отвлекая Worker'а.
* Цель: Минимизировать общее время выполнения всех задач (Makespan) и свести простой процессоров к нулю.
